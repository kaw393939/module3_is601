# Module 3: Python Control Structures and CSV Files

## Module Overview
This module introduces students to Python control structures such as loops and conditionals, and techniques for reading and writing CSV and JSON files. Students will develop a Python project that reads a CSV file and dynamically generates an HTML table. By the end of this module, students will be proficient in implementing control structures in Python, manipulating CSV and JSON files, generating HTML content, applying error handling strategies, and writing tests using pytest.

### Why Control Structures and CSV Files?
Control structures like loops and conditionals are fundamental to programming, enabling conditional execution and iteration of code blocks. CSV and JSON are ubiquitous formats for data storage and interchange. Mastery of these formats is crucial for data processing, web development, and automation tasks in modern software development.

### Submit

**Activity Type:** Advanced Hands-on Assignment  

**Activity Title:** CSV to Dynamic HTML Table Project  

**Grading Type:** Points  

**Submission Instructions:** Submit a link to your GitHub repository containing the project.

**Instructions:** 
- Create a new Git repository locally and a corresponding repository on GitHub.
- Set up a Python project in VSCode with a well-structured directory layout.
- Create and activate a virtual environment for your project.
- Develop a Python script that reads a CSV file and generates a dynamic, responsive HTML table:
  - The CSV header should become the HTML table header with sorting functionality.
  - The CSV rows should become the HTML table rows with alternating row colors.
  - Implement pagination if the CSV file contains more than 100 rows.
- Ensure your script handles various CSV formats and includes comprehensive error handling:
  - Implement and compare both LBYL and EAFP error handling strategies.
- Write extensive unit tests using pytest to verify the functionality of your script.
- Develop integration tests using pytest to validate the entire workflow.
- Implement proper logging throughout your application.
- Create comprehensive documentation, including a README.md file with setup and usage instructions.
- Push your code to GitHub and ensure it follows best practices for code organization and documentation.
- Configure GitHub Actions to run your tests automatically on each push to the repository.

**Grading Expectations:** 
- Completeness and accuracy of the CSV to HTML table project
- Proper use of advanced control structures and efficient file handling in Python
- Correct implementation and comparison of LBYL and EAFP error handling strategies
- Comprehensive testing with pytest for both unit and integration tests
- Well-structured code with proper documentation and logging
- Adherence to Python best practices and coding standards
- Successful setup of GitHub Actions for automated testing

**Alignment:** 
- Navigate and execute advanced commands in a Linux environment
- Create, manipulate, and analyze CSV files using Python
- Implement and utilize best practices for software development security
- Write and execute comprehensive unit tests and integration tests for Python applications using pytest
- Apply error handling strategies to improve code robustness
